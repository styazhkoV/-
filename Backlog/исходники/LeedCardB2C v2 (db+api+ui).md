# Техническое задание: Модуль «CRM Лиды B2C» (v2.6)

## 1. Описание модуля

Модуль предназначен для управления полным жизненным циклом B2C-лидов (физические лица), поступающих из **Online Shop** (сайты kcell.kz, activ.kz) и подразделения  **Telesales** . Основной фокус модуля — высокая скорость обработки заявок, юридическая верификация через биометрию и автоматизация процессов до момента оплаты.

**Основные характеристики:**

* **Микросервис:** `sapa-crm-kcell-client`.
* **Типы лидов:** Полные (самостоятельное заполнение на сайте) и частичные (созданные Telesales).
* **Стек:** Java (Spring Boot), React, PostgreSQL (схема `client`).
* **Ключевые интеграции:** Биометрические сервисы, Облачная ЭЦП (вендоры), Nexign (биллинг), MinIO (хранение документов).

## 2. Маппинг полей по этапам воронки

Все первичные и внешние ключи используют тип  **bigint** . Персональные данные B2C изолированы в таблице расширения `client.clients_b2c`.

### Этап 1: ACQUAINTANCE (Идентификация и Профиль)

**Цель:** Однозначная идентификация физического лица по ИИН и сбор контактных данных.

| **Элемент UI**       | **Таблица в БД** | **Поле в БД** | **Тип** | **Бизнес-логика**                                    |
| --------------------------------- | -------------------------------- | -------------------------- | ---------------- | ---------------------------------------------------------------------- |
| ИИН                            | `client.clients`               | `bin_iin`                | `varchar(12)`  | Мастер-идентификатор. Уникальность 1:1. |
| Фамилия                    | `client.clients_b2c`           | `last_name`              | `varchar`      | Обязательное поле.                                     |
| Имя                            | `client.clients_b2c`           | `first_name`             | `varchar`      | Обязательное поле.                                     |
| Мобильный номер     | `client.clients_b2c`           | `phone`                  | `varchar`      | Формат: +77XXXXXXXXX.                                            |
| Язык обслуживания | `client.clients`               | `language_id`            | `bigint`       | Справочник: RU, KZ, EN.                                      |

### Этап 2: NEEDS (Корзина заказа)

**Цель:** Выбор устройства, тарифного плана или аксессуаров.

| **Элемент UI** | **Таблица в БД** | **Поле в БД** | **Тип** | **Бизнес-логика**                      |
| --------------------------- | -------------------------------- | -------------------------- | ---------------- | -------------------------------------------------------- |
| Продукт / Тариф | `client.lead_items`            | `product_id`             | `bigint`       | Справочник продуктов.                 |
| Тип позиции       | `client.lead_items`            | `item_type`              | `varchar`      | DEVICE / SERVICE / ACCESSORY.                            |
| Цена                    | `client.lead_items`            | `unit_price`             | `numeric`      | Автозаполнение из прайса.          |
| Сумма заказа     | `client.leads_b2c`             | `total_amount`           | `numeric`      | Авторасчет суммы всех позиций. |

### Этап 3: VERIFICATION (Юридическая чистота)

**Цель:** Подтверждение личности и подписание договора.

| **Элемент UI**       | **Таблица в БД** | **Поле в БД** | **Тип** | **Бизнес-логика**                          |
| --------------------------------- | -------------------------------- | -------------------------- | ---------------- | ------------------------------------------------------------ |
| Биометрия (Статус) | `client.leads_b2c`             | `biometric_status_id`    | `bigint`       | Интеграционный статус (Success/Fail).    |
| Облачная ЭЦП           | `client.leads_b2c`             | `cloud_sign_id`          | `bigint`       | Ссылка на транзакцию подписания. |
| Скоринг (Доходы)     | `client.clients_b2c`           | `income_amount`          | `numeric`      | Требуется для рассрочки/кредита. |

## 3. Описание методов API

Согласно спецификации `api-kcell.json` и логике `LeedCardB2C v2`.

### 3.1. Управление лидами

* **POST `/api/v1/leads/b2c`**
  * **Описание:** Регистрация нового B2C-лида.
  * **Payload:** `{ "bin_iin": "...", "source_id": bigint, "items": [...] }`
  * **Action:** Создает записи в `leads_b2c` и `lead_items`, запускает таймер SLA.
* **GET `/api/v1/leads/b2c/{id}`**
  * **Описание:** Получение данных лида, включая профиль из `clients_b2c` и статус биометрии.

### 3.2. Дополнительные действия

* **PATCH `/api/v1/leads/{id}/verification/biometrics`**
  * **Описание:** Обновление статуса биометрической проверки от внешнего провайдера.
* **GET `/api/v1/dictionaries/rejection-reasons`**
  * **Описание:** Получение списка причин отказа (например, "Не проходит по доходам").

### 3.3. Аудит и История

* **GET `/api/v1/leads/{id}/audit`**
  * **Описание:** Получение истории изменений (кто, когда и какое поле изменил).

## 4. Бизнес-правила

### 4.1. Автоматическое распределение (Round Robin)

1. **Триггер:** Поступление лида из Online Shop или ручное создание в Telesales без указания ответственного.
2. **Логика:** Система выбирает сотрудников из таблицы `users.users` со статусом `is_online = true` и ролью `B2C_OPERATOR`.
3. **Выбор:** Назначается оператор, который дольше всех не получал лидов (минимальный `last_assigned_at`).
4. **Фиксация:** Заполнение поля `operator_id` и `assigned_at` в `leads_b2c`.

### 4.2. Жесткий SLA Watchdog (15 минут)

1. **Условие:** Лид находится в статусе `NEW`.
2. **Контроль:** Если `now() - assigned_at > 15 минут`, сервис `Task/Watchdog` устанавливает флаг `is_sla_breached = true`.
3. **Эскалация:** WebSocket-уведомление мгновенно отправляется в UI Супервайзера.

### 4.3. Особенности верификации

* Переход на этап **SALE** (Продажа) заблокирован до получения статуса `SUCCESS` от сервиса биометрии.
* При выборе типа оплаты "Рассрочка" поля `income_amount` и `income_source_id` становятся обязательными для заполнения.
