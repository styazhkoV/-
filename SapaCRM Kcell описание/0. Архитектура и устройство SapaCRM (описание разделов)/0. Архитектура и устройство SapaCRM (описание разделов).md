* **Глава 1: Бизнес-контекст и Глобальный ландшафт (C4 System Context)**
  * Роль SapaCRM в экосистеме Kcell (обслуживание B2C и B2B).
  * Взаимодействие с внешним миром (Сайты, ESB, Nexign, SAP, Avaya, SAO).
  * Ограничения и этапность релизов (проблема разрыва во времени между внедрением CRM и DWH/SAP).
* **Глава 2: Подсистема IAM и Безопасность (Как устроен доступ)**
  * Сквозная авторизация: Active Directory -> Keycloak -> JWT -> SapaCRM.
  * Модель данных RBAC (Схема `users`: API, Roles, Permissions).
  * Инициализация микросервиса `user` (Разбор инцидента с `PKIX path building failed` и `cacerts`).
* **Глава 3: Ядро системы — Доменная модель "Карточка Клиента"**
  * Обзор схемы `client` (Клиенты, Компании, Договоры, Продукты).
  * Сложные бизнес-связи: Маршрутизация сотрудников, иерархия департаментов.
  * Архитектурный компромисс: разделение таблиц `contacts` и `authorized_persons`.
* **Глава 4: Управление бизнес-процессами: Лиды, Активности и SLA**
  * Омниканальный сбор лидов (Импорт, Web, SAO).
  * Модули `task` и `notification`. Алгоритмы маршрутизации (SME, SA, LA, Telesales).
  * *Примечание: для этой главы критически не хватает статусной модели Лида.*
* **Глава 5: Интеграционный слой и Хранилище документов**
  * Устройство сервиса `data` (Импорт данных): лимиты, degraded mode, парсинг Excel.
  * Сервис `doc`: взаимодействие с MinIO (бакет `kcell-documents`).
  * Асинхронное взаимодействие с тикет-системой SAO.
* **Глава 6: Инфраструктура, Развертывание и Конфигурация**
  * Топология Kubernetes-кластера (Helm-чарты микросервисов).
  * Схема `management` (Динамическое управление справочниками через In-memory кеширование).
