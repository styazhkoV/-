Задача: Дорабтка Campaign Management + фильтрация

Описание:
Необходимо привести поля фильтрации и отображаемые колонки в разделе Campaign Management (Сегменты) к аналогии с разделом "Клиенты", а также реализовать корректный поиск по этим полям.

1. Поля фильтрации:

- client_type  
  → источник: справочник STRUCTURE  
  → тип: множественный выбор (multi-select, массив значений)

- name  
  → тип: текстовое поле  
  → поиск по наименованию/ФИО

- bin_iin  
  → тип: текстовое поле  
  → поиск по БИН/ИИН

- contact_phone  
  → тип: текстовое поле  
  → поиск по номеру телефона

- region  
  → тип: текстовое поле  
  → поиск по региону

- manager  
  → источник: список пользователей  
  → тип: dropdown (single/multi по согласованию)

2. Колонки в таблице:

Необходимо добавить/привести к клиентскому модулю следующие колонки:

- Тип клиента (client_type)
- Клиент / Наименование (name)
- БИН/ИИН (bin_iin)
- Контактный номер (contact_phone)
- Регион (region)
- Менеджер (manager)

(если уже есть — проверить корректность отображения и соответствие данным)

3. Поведение:

- Все поля должны участвовать в фильтрации (backend + frontend)
- Поддержка комбинированного поиска (несколько фильтров одновременно)
- Значения должны корректно передаваться в API
- Результаты обновляются по кнопке "Применить"/"Поиск"
- Фильтрация должна совпадать с логикой модуля "Клиенты"

4. Дополнительно:

- Проверить соответствие названий полей API и UI
- Проверить маппинг client_type → STRUCTURE
- Для manager — использовать актуальный список пользователей
- Обработать null/пустые значения
- Проверить пагинацию с учетом фильтров

Результат:
В Campaign Management реализованы колонки и фильтрация, аналогичные модулю "Клиенты", с корректной работой поиска по всем полям.
